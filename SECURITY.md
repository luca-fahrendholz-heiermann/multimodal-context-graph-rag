# Security Policy

## Supported usage
This repository is a portfolio/demo project and is maintained on a best-effort basis.

## Reporting a vulnerability
Please do **not** open public issues with sensitive details.
Instead, contact the maintainer privately and include:
- a short description,
- reproduction steps,
- potential impact.

## Security principles in this repo
- No real secrets or API keys are committed.
- Demo mode works without provider credentials.
- Logs should not include raw sensitive payloads or full personal data.
